Debian, RHEL, and SUSE

RHEL - Red Hat Enterprise Linux also includes CentOS, Fedora, Oracle Linux. Fedora has additional functionality not included in red hat

  • Fedora serves as upstream testing platform for RHEL
  • CentOS is a close clone of RHEL, while Oracle Linux is a copy of RHEL with some changes
  • uses 'yum' and 'dnf' RPM-based yum package managers
  • RHEL is widely used by enterprises that host their own systems

SUSE Family

  • SUSE Linux Enterprise Server (SLES) is the upstream for openSUSE
  • uses 'zypper' package manager (RRM-based)
  • includes YaST (Yet Another Setup Tool) app for sys admin purposes
  • SLES is widely used in retail

Debian Family
Ubuntu is the upstream for Linux Mint and a number of other distros. commonly used on server and clients.
Debian provides the largest and most complete repository of any Linux distro.

  • Debian is upstrean for Ubuntu and Ubuntu is upstream of Mint, Kubuntu, and many others
  • uses 'apt' package manager (which is DPKG-based, NOT RPM-based)
  • widely used in cloud deployment
  • GNOME under the hood

Terminology:


  • kernel - the lik between hardware and applications, the brain of the Linux OS
  • Distro (distrobution) - Red Hat, Ubuntu, Fedora, etc...
  • bootloader - boots the OS (2 examples: GRUB and ISOLINUX)
  • service - program that runs as a background process (ex: named, httpd, ftpd, dhcpd)
  • filesystem - method to store and organize files (ex: Fat, ext3, ext4, XFS, NTFS)
  • x windows system - toolkit and protocols to build GUI (ex: desktop environments like GNOME or KDE)
  • command line (CLI) - interface for typing commands on top the OS
  • shell - is the Command Line Interpreter that interprets instructions entered in the CLI and informs the OS to perform the tasks and commands (ex: bash)

summarize the key concepts covered:

  • Linux borrows heavily from Unix
  • Features and Services are accessed through files and file-like objects
  • fully mult-tasking, multi-usser operating system, built in networking and service process known as daemons
  • developed by a collaboration from all over the internet
  • common terms: kernel, distrobution, boot loader, service, filesystem, x window system, desktop environment, and command line
  • a full distribution consist of the kerner, plus a number of other software tools for file-related operations, user management, and package management
